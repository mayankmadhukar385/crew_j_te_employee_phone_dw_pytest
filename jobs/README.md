# Jobs Directory

One folder per migrated DataStage job. Generated by **Step 3**.

## Structure per job:

```
{job_name}/
├── __init__.py
├── transformations.py              ← Job-specific derivation logic (ONLY custom code)
├── run_pipeline.py                 ← Pipeline orchestrator (imports from framework)
├── deploy_config.sql               ← INSERT statement for etl_job_config (run once)
└── tests/
    ├── conftest.py
    ├── test_transformations.py     ← Unit tests
    ├── test_integration.py         ← End-to-end with synthetic data
    └── fixtures/
        ├── synthetic_data.py       ← DataFrame factories
        └── expected_outputs.py     ← Expected results for assertions
```

## Rules:

- **NEVER** duplicate framework code — only imports from `framework/`
- **NEVER** hardcode table names or column names — read from config (Delta table)
- **ALWAYS** use native PySpark functions (F.trim, F.when, etc.) — no wrapper classes
- **ALWAYS** use `DatabricksJobLogger` for all logging including record rejects
- **ALWAYS** use `AuditLogger` for pipeline start/end tracking
- **ALWAYS** use `DeltaManager` for all target writes (SCD or plain modes)
- **ALWAYS** use `ConfigManager` to read config from the Delta table
- **ALWAYS** register test source data as temp views (`source.type: "test_dataframe"`)
